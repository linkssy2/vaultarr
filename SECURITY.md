# Security Policy

## Reporting a vulnerability

Please do not open a public issue for security-sensitive reports.

If you find a vulnerability, contact the maintainers privately through the preferred contact method listed on the repository page.

## Supported versions

The latest stable release receives security attention first.

## Notes

Vaultarr may interact with local files, provider APIs, cached media, manuals, backup archives, and configured library folders. Be careful when reporting issues that include local paths or private provider keys.
