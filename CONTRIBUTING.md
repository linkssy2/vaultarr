# Contributing to Vaultarr

Thanks for helping improve Vaultarr.

## Before opening an issue

Please include:
- Vaultarr version.
- Operating system or Docker environment.
- What you expected to happen.
- What actually happened.
- Screenshots or logs if useful.

## Development workflow

Recommended branch flow:
- `main` is stable release code.
- `develop` is for upcoming work.
- Feature branches should be named like `feature/media-cache` or `fix/sidebar-layout`.

## Pull requests

A good pull request should:
- Focus on one feature or fix.
- Avoid unrelated formatting churn.
- Include a clear description.
- Mention any new settings, migrations, or provider behavior.

## Project philosophy

Vaultarr treats games as digital artifacts. Features should support preservation, collection clarity, and a polished self-hosted experience.
