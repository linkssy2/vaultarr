# Vaultarr 2.0.0 Alpha 38 — Library-First Expanded Cards

## Soundtrack

- Presents owned and acquired tracks at the top of the expanded card's Soundtrack tab.
- Makes each listed track directly playable through Vaultarr's existing local audio player.
- Keeps the track count, active-track state, metadata, file size, and playback controls in one primary library panel.
- Groups file import and permission-confirmed direct audio acquisition under **Add owned music**.
- Groups catalog scanning, external previews, search shortcuts, and manual source links under **Find soundtrack sources**.

## Manuals

- Presents the saved manual reader and its open, download, source, and removal actions first.
- Groups provider search and manual-link entry under **Find or add a manual**.

## Gallery

- Presents cached media, the active preview, cache count, and media actions first.
- Keeps provider search directly beneath the saved library when requested.
- Moves cover selection, provider alternatives, and upload into **Manage the library cover**.

## Trailer

- Presents the saved trailer stage and its primary actions first.
- Consolidates recommendations, external searches, and manual URL entry under **Find or replace the trailer**.
- Removes the obsolete separate advanced-trailer panel and its unused styling.

## Acquisition

- Presents the saved source and local-copy status as the primary acquisition record.
- Separates catalog discovery, permission-confirmed direct-file importing, and existing local-copy attachment into three focused secondary sections.
- Keeps a saved authorized download link reachable after reopening a game without changing validation or download behavior.
- Corrects saved-record JSON hydration so existing acquisition sources no longer disappear behind an HTTP 500 response.

## Preserved systems

Manual, Gallery, Trailer, Soundtrack, and Acquisition storage and playback routes, existing scanners, authorized-download validation, Museum card dimensions and interactions, Museum Scan, Orbital Core, sidebar navigation, authentication, Museum Backup, and Milestone systems are unchanged.
